Command Line and Launchers

SnakeSh can be started with snakesh, python -m snakesh, or the compatibility alias securepython. The same settings, theme, secrets configuration, and startup master-password gate apply no matter which launcher path you use.

Main Application Launches

  • snakesh: start the main GUI normally.
  • snakesh <import_file.ssx>: start SnakeSh and immediately open an exported SnakeSh bundle for import.
  • snakesh --install-desktop: install or repair the Linux AppImage desktop entry and launcher integration, then exit.
  • snakesh --uninstall-desktop: remove the Linux AppImage desktop entry, launcher integration, and managed tool launchers, then exit.
  • snakesh --remove-tool-launchers: remove managed standalone tool launcher entries, then exit.
  • snakesh --debug-level {info,debug,trace}: enable a per-run diagnostics log for the current launch.
  • snakesh --debug-level debug --debug-log-file /path/to/log: write the diagnostics log to a specific file instead of the default debug-log folder layout.

--debug-log-file requires --debug-level. Without an explicit log path, diagnostics logs are written into SnakeSh's managed debug-log directory tree for the current platform.

Standalone Tool CLI

The tool mode launches the same standalone dialogs that appear under the main Tools menu. Run snakesh tool --help to print the current command reference.

  • snakesh tool list: print the registered tool keys in launch order.
  • snakesh tool resource_monitor: launch Resource Monitor.
  • snakesh tool network_inspector: launch Network Inspector.
  • snakesh tool whois: launch Whois.
  • snakesh tool asn_lookup: launch ASN Lookup.
  • snakesh tool dig: launch Dig.
  • snakesh tool traceroute: launch Traceroute.
  • snakesh tool ping: launch Ping.
  • snakesh tool ip_scan: launch IP Scan.
  • snakesh tool mtu_calculator: launch MTU / MSS Calculator.
  • snakesh tool file_hash: launch File Hash.
  • snakesh tool oui_lookup: launch OUI Lookup.
  • snakesh tool web_server: launch Web Server.
  • snakesh tool syslog_snmp_monitor: launch Syslog / SNMP Monitor.
  • snakesh tool subnet_calculator: launch Subnet Calculator.
  • snakesh tool password_generator: launch Password Generator.
  • snakesh tool diff: launch Diff Tool.
  • snakesh tool help: launch the standalone Help window.

snakesh tool help launches the Help tool itself. Use snakesh tool --help when you want the CLI usage text instead. Every registered tool also supports a tool-specific help view such as snakesh tool resource_monitor --help.

All standalone tools accept --debug-level {info,debug,trace} and --debug-log-file PATH. These debug flags may be placed before the tool key, as in snakesh tool --debug-level debug resource_monitor, or after the tool key, as in snakesh tool resource_monitor --debug-level debug. If both positions are used, the tool-specific value after the tool key takes precedence.

Ping Prefill Options

Only the standalone Ping tool accepts extra non-debug CLI arguments. These values prefill the GUI; they do not automatically start a ping run.

  • snakesh tool ping --packet-size 1472: open Ping with the packet-size field prefilled.
  • snakesh tool ping --ipv6: open Ping with IPv6 mode enabled.
  • snakesh tool ping --packet-size 1452 --ipv6: combine both prefill options.

Internal Helper Modes

SnakeSh also exposes internal helper modes because the main app launches some privileged or isolated helper processes by re-running itself. These flags are real command-line options and appear in --help, but they are meant for SnakeSh-managed child processes rather than normal manual use.

  • --web-server-helper INSTANCE_DIR: run the Web Server worker for an existing helper session directory.
  • --network-inspector-ports-helper SESSION_DIR: run the privileged Network Inspector ports collector for an existing helper session directory.
  • --mtr-helper SESSION_DIR: run the native traceroute or MTR helper for an existing helper session directory.
  • --syslog-snmp-monitor-helper PROFILE_ID: run the Syslog / SNMP Monitor collector for an existing monitor profile.

Launcher Entries

On Linux, the main app can install desktop integration and standalone tool launchers from Settings. On Windows and macOS, SnakeSh can manage per-user launcher entries for standalone tools from the same Manage Tool Launchers... workflow. Use --remove-tool-launchers for cross-platform cleanup without changing SnakeSh data. Standalone tools share the same saved settings, theme, and optional tool-launch master-password gate as the main app.
