Tool: Web Server

The Web Server tool launches a GUI-managed helper that can either serve static files or run a single-upstream reverse proxy. It is designed for quick testing and local hosting, not for deploying a full general-purpose application server stack.

Profiles

  • Profile: drop-down list of saved web server configurations. The current selection is remembered and restored the next time you open the tool.
  • Load: applies the selected saved profile to the form without starting the server. If a different profile is loaded while a server is still active, SnakeSh warns first and stops the current helper before switching to the new profile.
  • Save As...: saves the current form fields as a new named profile.
  • Update: overwrites the selected profile with the current form values.
  • Rename...: changes the selected profile name.
  • Delete: removes the selected profile after confirmation.

Profiles are stored in SnakeSh settings so they remain available across restarts and can be included in normal settings export or import flows.

Fields

  • Bind Address: choose a detected interface address, a wildcard bind such as 0.0.0.0 or ::, or switch to Custom... to type a manual IPv4 or IPv6 address. Defaults to 127.0.0.1.
  • Port: TCP port to listen on. The suggested value follows the current mode and TLS selection until you override it manually.
  • Mode: chooses between Static Files and Reverse Proxy.
  • Document Root / Index Page / Directory Listing: static-file settings shown only in Static Files mode.
  • Upstream URL: reverse-proxy origin used in Reverse Proxy mode.
  • Path Prefix, Strip Prefix, Preserve Host, X-Forwarded-*, Upstream TLS Verify, WebSocket Support, timeout fields, and Extra Request Headers: structured reverse-proxy controls shown only in Reverse Proxy mode.
  • TLS Mode: chooses HTTP Only, Manual Certificate, Self-Signed Certificate, or Certbot (HTTP-01).
  • Certificate File, Key File, and optional Intermediate / Chain File: manual-TLS inputs.
  • Certbot Executable, Primary Domain, Additional Domains, Contact Email, HTTP-01 Port, and Staging: guided certbot inputs shown only in certbot mode.

Buttons

  • Start: validates the configuration, writes the helper config, and launches the helper process.
  • Stop: requests server shutdown through the helper stop signal.
  • Open in Browser: opens the currently reported server URL in the default browser when the server is running.
  • Open Current Log: opens the archived log file for the current or most recent run.
  • Open Log Folder: opens the web-server log archive folder.
  • Clear Log View: clears the visible log pane only.

Runtime Display

  • URL: shows the current helper-reported URL or (not running).
  • Log output: shows appended helper logs.
  • Status line: reports validation problems, launch state, running state, and stop state.

Operational Notes

  • Unix-like systems may require GUI elevation for privileged listener ports or privileged certbot challenge ports.
  • Certbot checks run only when you click Start. SnakeSh does not renew certificates in the background or while the tool is stopped.
  • Archived web-server logs are retained and pruned using the web-server log cleanup settings in the main Settings dialog.
  • The selected saved profile and the tool's log/form splitter position are restored the next time you open the dialog.
  • Saved profiles can contain incomplete or not-yet-valid values. Validation still happens only when you click Start.
  • Closing the dialog while the server is running requests helper shutdown.
