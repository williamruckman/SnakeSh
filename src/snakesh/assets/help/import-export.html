Import and Export

SnakeSh includes first-party import and export so you can back up your environment, move it to another machine, or share a sanitized subset with teammates.

What You Can Export

  • Sessions only
  • Settings only
  • Combined sessions and settings
  • A selected subset of sessions instead of the entire library

Settings exports include the normal application preferences such as appearance, workspace profiles, fast commands, web server profiles, Syslog / SNMP monitor profiles, and password-generator defaults.

For the Syslog / SNMP Monitor, the exported profile metadata includes saved listener settings, filters, visible columns, splitter layout, and display-timezone preference.

Export Protection

Exports can be written without a password, but encrypted export bundles are available when you need to store or share the file more safely. Use password-protected exports for anything beyond short-lived local backups.

What Is Not Bundled

Saved session passwords and backend auth values such as service tokens, vault tokens, or CLI unlock secrets are not embedded in first-party exports because those values stay in the OS keyring or the selected secrets backend. Captured Syslog / SNMP monitor history, pending alert rows, and archive files also remain local application data instead of being packed into first-party exports. After importing on another machine, reconnect the backend or re-enter any required secrets there.

Import Modes

Settings import replaces the current application settings after confirmation. Session import supports merge and overwrite modes so you can decide whether imported sessions should coexist with or replace current ones.

Third-Party Import

The Third Party Import workflow can ingest SecureCRT XML, OpenSSH configuration files, and PuTTY session definitions. Third-party export is intentionally not provided, which keeps SnakeSh from trying to guarantee compatibility with every external format on write.

Practical Advice

  • Create a password-protected export before large session-library refactors.
  • Use selective export when sharing only a specific customer or environment folder.
  • Prefer merge mode during trial imports so you can inspect the result before replacing existing data.
