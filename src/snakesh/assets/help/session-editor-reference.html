Session Editor Reference

This page is the field-by-field reference for the Session Editor. SnakeSh hides and resets some fields automatically when the selected protocol does not support them, so the same dialog behaves differently depending on whether you are editing SSH, SFTP, RDP, VNC, NoMachine, Telnet, or Serial sessions. Local Shell tabs are launched separately from the main window and are configured globally in Settings rather than through the Session Editor.

Always-Visible Core Fields

  • Name: display name of the saved session in the session tree and workspace labels.
  • Protocol: session type. Changing this immediately updates the visible fields and default port.
  • Folder: session-library folder path. Defaults to Default if left blank when saving.
  • Tags (comma-separated): free-form tag list used for filtering and organization.
  • Notes: stored free-form notes for the session.

Common Network Connection Fields

  • Host: network hostname or address. Hidden for Serial sessions. For Serial, the saved host field is reused to store the selected port path or COM port.
  • Username: username for network protocols that use it. Hidden for Serial.
  • Port: network port number. Hidden for Serial. Automatically resets to the protocol default when the protocol is changed.
  • Domain (RDP): Windows domain field shown only for RDP sessions.

SSH and SFTP Fields

  • Default Local Folder (SFTP): preferred starting folder on the local machine for SSH or SFTP file-transfer workflows.
  • Default Remote Folder (SFTP): preferred starting folder on the remote host for SSH or SFTP file-transfer workflows.
  • Use key auth: enables key-based authentication for SSH or SFTP sessions.
  • Private Key Path: path to the private key file. The Browse button opens a file picker filtered for common private-key file types.
  • Public Key Path: path to the matching public key file. The Browse button opens a file picker filtered for .pub files.
  • Enable X11 forwarding (SSH/SFTP): enables X11 forwarding for the SSH family.
  • Use legacy SSH compatibility mode (older endpoints): enables compatibility-oriented SSH behavior for older servers or network devices.
  • Enable SSH keepalive packets (30s interval): enables keepalive traffic to reduce idle disconnects.
  • SSH Tunneling...: opens the SSH tunneling editor described later on this page.

Password and Credential Fields

  • Save password in OS keyring: stores the password in the configured secrets backend or OS keyring for supported protocols. The checkbox is shown only for SSH, SFTP, RDP, and VNC.
  • Password (optional): password entry field. Shown only for protocols that support saved-password storage through the current editor model.
  • Show / Hide button beside Password: reveals the current password field and, when editing a session with saved credentials, can load the stored password for display so it can be reviewed or replaced.

Per-Session Terminal Color Overrides

  • Override global terminal colors for this session: enables per-session terminal colors.
  • Terminal Background Override: background color used when per-session override is enabled.
  • Terminal Foreground Override: foreground color used when per-session override is enabled.
  • Terminal Color Preview: live preview of the selected terminal override colors. When overrides are disabled, the preview indicates that the global colors remain in effect.
  • Pick buttons beside the color fields: open the color picker.

Shell Banner Fields

  • Shell Banner Message: optional plain-text message shown above the connected terminal. Visible only for SSH, Telnet, and Serial sessions.
  • Shell Banner Font Color: text color used by the shell banner message.
  • Blink shell banner while connected: toggles a blinking warning banner. If the banner message is blank, the color and blink settings are ignored.

RDP and VNC Display Fields

  • Resolution (RDP/VNC): display resolution. Supports preset values such as 1024x768, 1280x720, 1920x1080, plus Default and Auto. The field is editable, so a custom resolution can be typed manually.
  • Launch in fullscreen: starts the remote desktop in fullscreen mode.
  • Allow dynamic remote resize: VNC-only option that allows remote resize when the selected VNC server and viewer support it.
  • Color Depth (RDP/VNC): display color depth. Choices are Default, 8-bit, 16-bit, 24-bit, and 32-bit.

RDP-Only Fields

  • Domain (RDP): Windows login domain.
  • Audio (RDP): audio routing policy. Choices are On this computer, On remote computer, and Do not play.
  • Open Mode: Linux-only RDP option that selects Open in tab or Open detached window. On non-Linux platforms the field is hidden and RDP is forced into detached mode.

NoMachine Fields

  • Enable audio streaming: enables remote audio streaming.
  • Mute audio on remote computer while connected: attempts to mute the remote side while audio is being redirected.
  • Auto-resize physical desktop: allows physical desktop resizing where NoMachine supports it.
  • Resize Mode (NoMachine): chooses Scale to fit or Viewport (scroll).
  • Link Quality (NoMachine): connection-quality scale from 0 to 9.
  • Video Quality (NoMachine): image-quality scale from 0 to 9.

Telnet Fields

  • Terminal Type (Telnet): advertised terminal type string. The field is editable and pre-populated with values such as xterm-256color, xterm, vt100, and ansi.
  • Connect Timeout (Telnet, s): connect-timeout value in seconds.
  • Use TLS (Telnet over SSL): enables TLS for Telnet connections.
  • Validate server certificate: certificate-validation toggle used when TLS is enabled.

Serial Fields

  • Serial Port: serial device selector. It is editable and includes a Refresh button to rescan ports. On Windows this includes COM ports; on Unix-like systems it scans common device-path patterns.
  • Baud Rate: editable baud-rate selector with common preset values from 300 through 230400.
  • Data Bits: serial data-bit width. Choices are 5, 6, 7, and 8 bits.
  • Parity: parity mode. Choices are None, Even, Odd, Mark, and Space.
  • Stop Bits: stop-bit count. Choices are 1, 1.5, and 2.
  • Flow Control: flow-control mode. Choices are None, RTS/CTS, XON/XOFF, and DSR/DTR.

Automation Controls

  • Enable automated scripting after connection (SSH/Telnet/Serial): enables post-connect automation for the supported terminal protocols.
  • Automated Scripting...: opens the automation editor. Visible only for SSH, Telnet, and Serial.
  • Automation summary label: shows a concise summary of the current automation steps so the session can be reviewed without reopening the automation dialog.

Automated Scripting Dialog

  • Add: create a new automation step.
  • Edit: edit the selected automation step.
  • Remove: delete the selected step.
  • Move Up / Move Down: reorder steps.
  • Step Type in the step editor: chooses Command, Sleep, or Expect.
  • Command: command text used by a Command step.
  • Seconds: delay used by a Sleep step.
  • Expect Text: required text match used by an Expect step.
  • Timeout (s): wait time for an Expect step.
  • On Timeout: Expect-step timeout behavior. Choices are Terminate Script and Continue Script.

SSH Tunneling Dialog

  • Dynamic (SOCKS) tab: manages SOCKS listeners forwarded through the SSH session.
  • Static tab: manages local and remote TCP forward rules.
  • Add / Edit / Remove buttons on both tabs: create, modify, or delete tunnel definitions.
  • Tunneling summary label: shows enabled and total dynamic/static tunnel counts after the dialog is closed.

Dynamic Tunnel Editor

  • Bind Host: local interface address for the SOCKS listener.
  • Bind Port: local TCP port for the SOCKS listener.
  • Enabled: whether the tunnel should be active when the session connects.

Static Tunnel Editor

  • Direction: chooses Local Forward or Remote Forward.
  • Bind Host: listening interface for the forward.
  • Bind Port: listening port for the forward.
  • Target Host: target endpoint host for the forward.
  • Target Port: target endpoint port for the forward.
  • Enabled: whether the rule is active when the session connects.

Protocol Visibility Rules

  • RDP shows domain, audio, and Linux-only open mode fields.
  • VNC shows remote-display fields and the dynamic-resize checkbox, but not the RDP-only domain or audio controls.
  • NoMachine shows the NoMachine quality and resize controls and always uses detached-viewer behavior.
  • Telnet shows terminal-type, timeout, and TLS controls.
  • Serial hides host-style network fields and shows device-line settings instead.
  • SSH and SFTP show key-auth, SFTP folder, X11, keepalive, legacy-compatibility, tunneling, and terminal-color-override controls.
  • Only SSH, Telnet, and Serial expose the automation and shell-banner controls.
  • Local Shell is not part of the Session Editor; configure its defaults in Settings and launch it from the main toolbar.

Save / Cancel

  • OK: saves the edited session definition.
  • Cancel: closes the editor without applying the current changes.
