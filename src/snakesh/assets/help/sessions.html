Sessions and Protocols

Create, edit, duplicate, bulk-edit same-protocol selections, and organize saved sessions from the main session library. The editor only shows fields that apply to the selected protocol, which keeps unrelated SSH, desktop, serial, and file-transfer settings out of the way.

Session Library Basics

  • Sessions can live inside folders and subfolders.
  • Folders can be renamed, removed, or used for bulk organization.
  • Sessions can be duplicated to create variants without rebuilding every field.
  • Same-protocol multi-select can open the bulk-edit workflow for shared field updates.
  • Multi-select support lets you connect to several sessions or move them together.

Common Session Fields

Most network sessions share a host, port, username, notes, tags, and protocol-specific display text. Some protocols expose additional fields such as RDP domain, terminal overrides, viewer resolution, NoMachine quality controls, or saved-password handling for SSH, SFTP, RDP, and VNC. Local Shell is launched from the main toolbar and configured globally in Settings rather than saved as a session type.

Terminal-Oriented Protocols

SSH

SSH sessions support key-based authentication, password fallback, host key trust prompts, optional X11 forwarding, port forwarding profiles, post-connect automation, and in-app terminal tabs. The terminal tab can launch a paired SFTP workspace from the tab context menu.

SFTP

SFTP sessions open the dual-pane file-transfer workspace. They can reuse saved SSH-style credentials and support default local and remote start folders.

Telnet

Telnet sessions run inside the native terminal workspace. Optional TLS, terminal type selection, and certificate validation controls are available when the target environment requires them.

Serial

Serial sessions expose baud rate, parity, stop bits, data bits, and flow control options. They use the same terminal view as network sessions so searchable scrollback, logging, and tab handling stay consistent.

Local Shell

Local Shell is a built-in toolbar action rather than a saved session protocol. It opens a local terminal tab inside the same workspace model as SSH, Telnet, and Serial tabs, and its default command/start directory are configured globally in Settings.

Remote Desktop Protocols

RDP

RDP sessions launch through platform tools. Linux supports in-tab and detached modes with xfreerdp. Windows uses mstsc. Certificate trust can be confirmed from a GUI prompt on Linux before the viewer starts.

VNC

VNC sessions use an installed viewer when available. SnakeSh applies saved display preferences when the selected viewer supports them.

NoMachine

NoMachine sessions integrate with nxplayer and use a detached-viewer workflow. They still appear in the session library and share the same saved metadata model, including audio, resize, and quality preferences.

Per-Session Convenience Features

  • Saved passwords can be stored through the configured secrets backend instead of only inside the encrypted session store.
  • Per-session terminal foreground and background colors can override the global theme.
  • SSH, Telnet, and Serial sessions can display a shell banner with custom color and optional blink above the terminal.
  • Automation flows can run after connect for terminal-based protocols.
  • SSH and SFTP sessions expose public-key installation helpers.
