File Transfer and SFTP

The SFTP workspace is a dual-pane file-transfer view with the local machine on the left and the remote host on the right. It is designed for operational transfer tasks rather than only raw file browsing.

Opening SFTP

  • Connect directly to an SFTP session from the session library.
  • Open an SFTP tab from an existing SSH terminal so the same session definition is reused.

Navigation Model

Each side of the workspace combines navigation and file-list panes. The local side reflects your current machine, while the remote side reflects the connected SFTP server.

Transfer Actions

  • Double-click remote files to download.
  • Double-click local files to upload.
  • Use right-click upload, download, and New Folder... actions for explicit direction control and directory creation on either side.
  • Use drag and drop to upload files into the remote pane.
  • Press F5 in an SFTP tab to refresh both the local and remote pane contents without changing directories.

Progress and Cancellation

Large transfers use a progress dialog that reports percentage, elapsed time, speed, and cancellation status.

Transfer Safety

Uploads and downloads can warn before overwriting existing files. For transfers with multiple conflicts, you can allow all overwrites for the current transfer from the confirmation dialog.

Delete Safety

Both local and remote delete operations are available from the context menu. A confirmation prompt is shown by default, and the global warning behavior can be adjusted in Settings.

Terminal Path Drops

Dragging local files into a terminal view or the bottom command bar inserts full local paths wrapped in shell-safe single quotes. This keeps spaces and apostrophes intact when building commands.

Default Start Folders

SFTP-capable session definitions can store preferred local and remote starting folders. This is helpful when a workflow always begins in a project directory, upload staging path, or remote deployment location.
