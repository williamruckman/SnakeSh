Tool: Traceroute

The Traceroute tool presents an MTR-style live hop table while repeatedly tracing the current path to a target. By default SnakeSh uses the platform's normal unprivileged traceroute command set. An optional fast native mode can pipeline probes across cycles for better performance, but it may require elevation on some platforms.

Fields

  • Target: hostname or IP address to trace.
  • Max hops: upper hop limit.
  • Timeout per probe: timeout for each per-hop probe in milliseconds.
  • Interval: pause between completed cycles.
  • Cycles: choose Run until stopped, pick a preset from 1 through 10, or choose Custom to enter another count.
  • Protocol: choose Auto, ICMP, or UDP. Auto is the default and prefers the most reliable unprivileged backend for the current platform.
  • Engine: enable Fast native probing to use the pipelined raw-socket backend. This mode can start the next cycle before slow hops from older cycles finish timing out, but it may prompt for elevation when the OS requires raw-socket access.
  • Resolve hostnames: controls reverse lookup behavior for responding hops.
  • Use IPv6: switches the trace to IPv6 mode.

Buttons and Table

  • Start: launches the helper and begins collecting hop statistics.
  • Stop: requests a clean stop for the active trace.
  • Reset Stats: clears all accumulated hop counters and starts fresh on the next run.
  • Graph: opens a live analysis window for the current trace with multiple chart views and export actions.
  • Copy Report: copies a plain-text summary plus the current hop table.
  • Hop table: shows Hop, Host, Address, Loss%, Sent, Recv, Last, Avg, Best, Worst, and StDev. Columns are resizable. Right-click a cell or header to copy a cell, row, selected rows, or a whole column. The default widths keep Last and Avg visible while still leaving room to expand host and address columns.

Graph Window

The graph window follows the active traceroute session and updates live while the trace is running. It keeps the current run's per-probe history only; starting a new trace or using Reset Stats clears both the table summary and the graph history.

  • Views: RTT Over Time, Latency Summary, Packet Loss, Jitter, Hop Heatmap, Worst Hops, Cumulative Path, and Reachability Trend.
  • Filters: hop multi-select, timeout inclusion, and cycle range controls let you narrow the displayed history without changing the underlying trace data.
  • Export Image: saves the current graph tab as PNG or SVG.
  • Export CSV: exports the filtered raw probe history for the active session, including timeout rows.

Operational Notes

SnakeSh still uses a helper process to aggregate repeated trace cycles. In the default mode it relies on the host system's normal unprivileged traceroute command set. Auto prefers ICMP-based tracert on Windows and UDP-style traceroute or tracepath on Unix-like systems. The optional fast native mode uses direct raw-socket probes instead, which lets later cycles start while older probes are still waiting on replies or timeouts.

Fast native mode may require elevation, depending on the platform and current permissions. Its cycle number means the probe wave launch order, not the moment when every hop in that cycle has already resolved. Loss and latency statistics still accumulate across the full session until the trace stops or you reset the session. Avg is the running average latency for successful replies in the current session. Reverse DNS is best-effort and may leave the Host column blank or equal to the numeric address when a name is unavailable.
