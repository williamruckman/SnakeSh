Tool: File Hash

The File Hash tool computes or verifies file digests on a worker thread so large files do not block the GUI.

Fields

  • Target File: file to hash or verify. Use Browse... to select it.
  • Algorithm: hashing algorithm. Supported entries come from SnakeSh's built-in list, including MD5, SHA1, SHA256, SHA384, SHA512, and BLAKE2b.
  • Generated Digest: read-only field populated after generation or verification.
  • Manual Verify: optional known digest pasted directly into the dialog.
  • Checksum File: optional checksum file path. Use Browse... to select it.

Buttons

  • Generate: computes the digest for the target file using the selected algorithm.
  • Verify: verifies the file. If Manual Verify has a value, that path is used first. Otherwise the checksum file path is used.
  • Clear: clears all editable fields except the selected algorithm and resets the details area.

Details Pane

The details pane shows either generation metadata such as file path, size, elapsed time, and digest, or verification metadata such as expected digest, actual digest, verification source, and matched checksum entry name.

Operational Notes

  • Verification requires either a manual digest or a checksum file.
  • Common checksum-file formats are supported by the backing service.
  • While a background task is running, Generate, Verify, and Clear are disabled.
