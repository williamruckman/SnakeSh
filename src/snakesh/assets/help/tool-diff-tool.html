Tool: Diff Tool

The Diff Tool compares two text buffers side by side. You can paste content directly into either pane or open files from disk, then review and merge differences without leaving SnakeSh.

Layout

  • Left and right headers: each side has Open... and Save controls plus an eliding path label that shows the current file when one is loaded.
  • Two editors: side-by-side plain-text editors with line numbers and no wrapping, suitable for pasted config snippets or full files.
  • Center gutter: shows visual connectors between changed blocks and exposes per-block copy buttons in both directions.
  • Scroll maps: narrow overview bars on both sides show where changes are located in the full document and stay aligned with scrolling.
  • Search row: a built-in search field can be shown when needed and can target either the left or right editor.
  • Status line: reports whether the files are identical or how many changed lines were detected.

Comparison Behavior

  • Differences are recomputed automatically after edits, with a short debounce so typing does not relaunch the diff on every keystroke.
  • Line-level changes are highlighted across both editors, and replace blocks also receive inline character-level highlights.
  • Scrolling is synchronized between the two panes so matching regions stay aligned while you inspect changes.
  • Both editors use the terminal font family and point size from SnakeSh settings so the diff panes match the rest of the application.
  • Very large files trigger a warning before opening, and files above the hard size limit are refused to avoid freezing the editor.
  • Files that look binary trigger a confirmation prompt before loading because they usually are not meaningful text diff targets.

Merge Controls

  • > and < buttons in the center gutter copy a single changed block from one side to the other.
  • Apply All →: replaces every differing block on the left with the right-side content.
  • ← Apply All: replaces every differing block on the right with the left-side content.
  • Save: writes the current editor text back to the existing file path or prompts for a destination if the side was not opened from a file.

Search

  • The search row is hidden by default; click the Search button to show or hide it, or press Ctrl+F to open and focus it.
  • Left and Right choose which editor the current search targets.
  • Press Ctrl+F to open the search row and select the current query.
  • ▲ and ▼ move between matches inside the selected editor; the match label shows the current index.
  • Case enables case-sensitive search.
  • Regex treats the input as a regular expression.
  • Press Escape or use the close button on the search row to hide it and clear the active search highlights.
