Tool: IP Scan

The IP Scan tool performs built-in TCP connect scanning against a hostname, a single IP address, or a CIDR range. It is designed for quick operational discovery without requiring an external scanner package.

Targeting

  • Hostname: SnakeSh resolves the hostname to one or more TCP-capable addresses and scans each resolved address.
  • Single IP: scans exactly that host.
  • CIDR: expands the network into hosts and scans each one, up to the current safety cap.

Port Sets

  • Common TCP 20: a smaller operationally useful set for fast checks.
  • Common TCP 100: a broader built-in port list for service discovery.
  • Custom Ports: enter comma-separated ports and ranges such as 22,80,443,8000-8100.

Controls

  • Scan: starts the TCP scan.
  • Stop: requests cancellation of the running scan.
  • Clear: clears both result tabs and resets the progress display.
  • Copy Selected and Copy All: export the current tab as tab-separated rows with headers.
  • Progress bar: tracks probe completion while the scan is running.

Results

  • Hosts shows each scanned host, best-effort resolved name, status, open-port count, and elapsed time.
  • Open Ports shows one row per open TCP port along with the best-effort service name.
  • Each tab has its own live Filter box so you can narrow visible rows without rerunning the scan.
  • Click the host address or open-port count for a host with discovered ports to switch into the Open Ports tab with that host pre-focused.
  • Use Clear Host Focus on the Open Ports tab to return from a host-specific drilldown to the full port list.

Notes

The scanner is TCP-only. A host with no reported open ports is not guaranteed to be offline; it simply means no open TCP ports were found within the configured scan set and timeout. Reverse name enrichment is best-effort and may be blank even when a host is reachable.
