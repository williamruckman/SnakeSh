Tool: Syslog / SNMP Monitor

The Syslog / SNMP Monitor is a GUI-managed collector for receiving, browsing, alerting on, and archiving syslog messages plus SNMP traps and informs. It is built as a non-modal operations sidecar, so you can leave it open beside the main SnakeSh workspace.

Profiles and Layout

  • Profiles: save, load, update, rename, and delete reusable monitor configurations from the top row.
  • What a profile stores: listener settings, bind address, SNMPv3 users, alert rules, retention policy, filter state, visible columns, display timezone, and dashboard/detail splitter layout.
  • Top-right Status Badge: shows Running, Starting, Stopped, or Error with high-contrast color coding while the status line below the tabs keeps the full runtime message.
  • Tab Layout: use Settings for listener and protocol setup, Monitor for filters and received events, Alerts for rules, Dashboard for charts, and Archive / Retention for local data controls.
  • Bind Address: uses the same pulldown pattern as the Web Server tool, including wildcard presets, discovered interface addresses, and a Custom... entry for manual IPv4 or IPv6 input.
  • Window Controls: the monitor window can be maximized for wide event tables and charts.

Listener Setup

  • Syslog UDP defaults to port 1514.
  • Syslog TCP defaults to port 1514.
  • Syslog TLS defaults to port 6514 and requires a certificate plus key before start.
  • SNMP Traps / Informs default to UDP port 1162.
  • SNMP Versions: v1, v2c, and v3/USM are configured from the same tool window. Community strings and SNMPv3 users are entered entirely through GUI dialogs.

On Linux and macOS, using privileged ports such as 514 or 162 can trigger a GUI elevation prompt. SnakeSh keeps the default listener ports non-privileged so normal use does not require that prompt, and the elevated helper is launched back into the same SnakeSh profile-data location so the GUI can continue showing status, events, and stop control normally.

Searching and Filtering

The Monitor tab combines a quick find field with a broader filter panel above the received event view. You can filter by time range, source, listener, protocol, transport, syslog severity or facility, hostname, app or tag, PID, message ID, SNMP version, security name, notification OID, enterprise OID, varbind text, alerted-only state, and live versus archived data scope.

Start and end timestamps expect ISO-style text such as 2026-03-31T15:30:00+00:00. Regex and case-sensitive search are optional toggles.

Table, Copy, and Export

  • Double-click an event to open it in a separate popup window with Fields, Raw Payload, and JSON tabs. The popup can also be maximized.
  • Display Timezone: choose UTC, System Local, or a specific IANA timezone from the pulldown above the event table. This changes only how times are shown in the GUI.
  • Copy the current cell, current row, selected rows, current column, or all visible rows.
  • Show or hide columns from the Columns menu. Common fields such as facility, severity, hostname, app/tag, PID, message text, notification OID, and varbind summary can all be displayed.
  • Export the current filtered result set as CSV or JSON.
  • Export only the selected rows when you want to hand off a narrow incident slice instead of the full filtered view.

CSV and JSON exports keep the stored timestamp values. Copy actions that read from the visible table use the currently selected display timezone.

Alerts

Alert rules can match syslog or SNMP events by severity threshold, source text, app or tag, trap OID, enterprise OID, or general message and varbind text. Each rule can show a popup, play a sound, or do both.

Instead of spawning a separate popup for every event, SnakeSh keeps current alerts in one live alert window that updates as new alerts arrive. Use Dismiss Alerts to clear the current list. Sound-enabled rules use the same bell-style sound path as the terminal bell so audio behavior stays consistent.

Dashboard

The Dashboard tab graphs the current filtered subset with event-rate, severity-distribution, top-source, top-app/tag, top-trap, and alert-over-time views. Severity slices and the top-source, top-app, and top-trap charts can be clicked to push values back into the Monitor tab filters. Chart colors follow the active SnakeSh theme so the dashboard stays readable in either dark or light layouts.

Retention and Local Storage

  • Hot Database Retention: how long live events remain in the local SQLite database.
  • Archive Retention: how long compressed archive files are retained.
  • Maximum Archive Size: total on-disk quota for archive files before oldest archives are purged.
  • Archive Rotation Threshold: archive maintenance threshold value kept in the profile for planning and future tuning.
  • Archive Now and Purge Now: available only while the collector is stopped so maintenance does not race the active writer.
  • Clear Database...: available only while stopped. It warns before deleting all live events, pending notifications, and archived files for the current profile.

Closing the monitor while the collector is still running shows a warning first. If you confirm, SnakeSh stops the collector before allowing the window or the main application to finish closing.

Monitor history is stored locally under SnakeSh's application data directory in a per-profile folder. Settings exports include the saved monitor profiles, but they do not bundle the captured event database or compressed archive history.
