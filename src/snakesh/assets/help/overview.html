SnakeSh Overview

SnakeSh is a multi-protocol remote access workspace that combines terminal sessions, file transfer, remote desktop launchers, local utilities, and encrypted session storage inside one Qt application.

What SnakeSh is best at: keeping a mixed SSH, SFTP, RDP, VNC, NoMachine, Telnet, Serial, and Local Shell workflow in one place without forcing you to bounce between separate tools.

Main Application Areas

Session Library

The left-side session tree stores your saved connections, folders, and organization metadata. Sessions can be tagged, duplicated, bulk-edited when the selection shares a protocol, renamed, moved between folders, or opened in bulk. The panel can stay visible, switch to auto-hide, or move into its own always-on-top floating window from the dropdown beside the search box.

Workspace

The center workspace holds terminal tabs, SFTP panes, remote viewer tabs, split layouts, detached windows, and the main window fullscreen workflow. Terminal-style tabs can participate in grouping, profiles, searchable scrollback, logging, shell-banner warnings, and reattach flows. Remote desktop viewers stay visually separate from terminal command injection so that SnakeSh does not guess at protocol-specific keystroke behavior.

Command Areas

SnakeSh has two distinct command mechanisms:

  • The bottom command bar is for active terminal tabs only.
  • Saved Fast Commands can target grouped terminal tabs or copy text to the clipboard when the active tab is an RDP, VNC, or NoMachine viewer.

Tools

The Tools menu launches detached standalone tool processes instead of opening the tool UI inside the main workspace process. This keeps a busy utility such as Resource Monitor or Ping from stalling the console tab and also lets tools stay open after the main SnakeSh window closes. Current tools include Resource Monitor, Network Inspector, Whois, ASN Lookup, Dig, Traceroute, Ping, IP Scan, MTU / MSS Calculator, File Hash, OUI Lookup, Web Server, Syslog / SNMP Monitor, Subnet Calculator, Password Generator, Diff Tool, and Help.

Settings and Security

The Settings dialog controls themes, colors, terminal behavior, main-window fullscreen behavior, Local Shell defaults, session and web-server log retention, secrets backends, desktop integration, per-tool launcher management, startup and tool-launch security, import, and export. Sensitive session data is encrypted at rest and password reuse can be delegated to a supported secrets backend.

Recommended Workflow

  • Create folders that reflect environments, customers, or roles.
  • Save sessions with protocol-specific details instead of re-entering connection settings.
  • Open an SSH tab for command work, then launch an SFTP tab from that saved SSH session when file transfer is needed.
  • Use tab groups and profiles for repeatable daily worksets.
  • Open the Help window or a standalone tool beside the main interface, or launch the tool directly with snakesh tool <tool_key>.
  • Open Command Line and Launchers from the Help index when you need the full CLI option reference, including debug flags, desktop integration commands, helper modes, and every registered tool key.

Platform Notes

Some capabilities depend on the operating system or installed external programs. Examples include xfreerdp for Linux RDP, platform VNC viewers, nxplayer for NoMachine, pysnmp for SNMP notification support, and OS-level elevation prompts when a web server or monitor listener must bind to a privileged port. The Troubleshooting section covers these differences in more detail.
