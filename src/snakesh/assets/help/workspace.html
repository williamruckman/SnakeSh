Workspace and Tabs

The workspace is where active connections live after you launch them from the session library. SnakeSh treats terminals, file transfer tabs, and remote viewers as related but distinct workspace types so each mode gets the controls it actually needs.

Session List Panel

  • The search box filters the saved session tree as you type.
  • The dropdown beside search switches between Show, Auto, and Float session-list modes.
  • In Auto mode the session list collapses until you hover the main splitter handle, and you can still drag that handle to resize the panel while it is revealed.
  • In Float mode the search box, session tree, and mode dropdown move into a separate always-on-top window. Closing that window returns the list to Show in the main workspace.

Split Panes and Tab Movement

  • SSH and SFTP tabs can move between horizontal and vertical split panes.
  • Tabs can be rearranged by drag and drop across panes.
  • This allows side-by-side work such as a Local Shell next to an SSH session, or an SSH tab next to its SFTP workspace.

Detached Windows

Terminal-style tabs can be detached into independent non-modal windows when you want to place them on another monitor or keep them visible while reorganizing the main workspace. SnakeSh now treats Local Shell tabs the same way as SSH, SFTP, and Telnet tabs for detach and reattach behavior.

Main Window Fullscreen

SnakeSh can switch the main window into borderless fullscreen using the configured fullscreen shortcut. Settings can also hide the top action bar and bottom command bar while fullscreen is active so only the session list and workspace stay visible.

Tab Context Actions

  • Rename Tab changes the visible tab label for the current workspace session.
  • Disconnect ends an active terminal session without closing the tab, so the current screen contents stay available.
  • Reconnect Session is available for closed reconnectable terminal tabs and reconnects them in place.
  • Open SFTP Tab appears on saved SSH terminal tabs, including closed SSH tabs in the tab strip, so file transfer can reuse the same session definition.
  • Group Manager and related group actions control fan-out behavior for terminal commands.

Tab Groups

Grouped terminal tabs receive the same typed command from the bottom command bar or from a Fast Command launch. Grouped tabs are marked in their title, and the group manager lets you rename groups or change membership without reconnecting the sessions.

Profiles

Profiles capture the current workspace state so it can be restored later. A profile can include:

  • Open sessions and tab order
  • Split pane layout
  • Detached windows and their geometry
  • Custom tab labels
  • Group assignments
  • Optional tool windows to start when the profile loads

You can also select a default startup profile to reopen a known workspace every time SnakeSh launches, and the Profile Manager can configure which tools should open with each saved profile.

Bottom Command Bar

The bottom command bar is intentionally terminal-only. It targets the active terminal tab or the active tab's group, and its Scrollback button opens the searchable history window for the active terminal tab without blocking terminal use. It does not send input to RDP, VNC, or NoMachine viewer tabs.
