Remote Desktops

SnakeSh supports RDP, VNC, and NoMachine sessions through integrated launch workflows. These sessions still live in the session library, but their runtime behavior is different from terminal tabs because desktop viewers do not share a universal input API.

RDP

  • Windows uses mstsc.
  • Linux uses xfreerdp.
  • Linux can run supported RDP sessions inside SnakeSh tabs or as detached viewers.
  • The Open Mode field appears only on Linux; non-Linux RDP is always detached.
  • Saved credentials and RDP domain values can be applied where supported.

VNC

VNC uses an installed platform viewer such as TigerVNC or compatible alternatives. SnakeSh detects available viewers and applies display preferences when the selected client supports them.

NoMachine

NoMachine uses nxplayer and follows a detached-viewer workflow. This keeps the session managed inside SnakeSh without pretending the viewer behaves like an embedded terminal. Saved sessions can also tune audio streaming, remote audio muting, physical-desktop auto-resize, resize mode, and link/video quality.

Display Preferences

RDP and VNC sessions can store options such as resolution, fullscreen, and color depth. These settings stay attached to the session definition so reconnects use the same viewing preference each time. NoMachine uses its own quality and resize controls instead of the shared RDP/VNC resolution fields.

Clipboard-Oriented Fast Commands

Saved Fast Commands behave differently on remote viewer tabs. Instead of trying to type into the remote desktop, SnakeSh copies the command text to the local clipboard and tells you to paste it manually inside the remote session. This keeps the behavior predictable across viewer implementations.

Platform Limits

Not every protocol supports in-tab rendering on every operating system. When a viewer must run detached, SnakeSh still manages the session entry and launch flow, but the actual desktop surface opens through the platform viewer process. See Platform Differences in the Help index for a direct Windows, Linux, and macOS comparison.
