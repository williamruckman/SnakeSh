Troubleshooting

This section covers common issues that come from platform differences, missing dependencies, or permission requirements rather than from a broken session definition.

Start Here for OS-Specific Behavior

If you are trying to understand why a feature behaves differently on Windows, Linux, or macOS, open the Platform Differences page from the Help index first. It explains which variations are expected and which ones usually indicate a real problem.

Viewer or Protocol Dependency Missing

Some protocols depend on external programs such as xfreerdp, mstsc, vncviewer, nxplayer, xauth, or ssh-keygen. SnakeSh checks for these tools on demand and warns when the required dependency is unavailable.

Web Server Permission Errors

Binding to privileged ports on Unix-like systems may require elevation. Use a non-privileged port such as 8080 when possible. If a privileged port is necessary, SnakeSh will try to request GUI-based elevation instead of expecting you to launch a shell manually.

Syslog / SNMP Monitor Will Not Start

Check the selected profile's listener settings first. Syslog TLS requires a valid certificate and key, and SNMP notification support requires the bundled Python runtime to include pysnmp plus pyasn1. On Linux and macOS, privileged ports such as 514 and 162 can also require a GUI elevation prompt before the collector can bind successfully. After you approve that prompt, SnakeSh expects the helper to report back into the same profile-data directory; if the monitor still does not move into a running state, recheck the privileged-port prompt result and any local policy restrictions around pkexec or administrator-privileged app launches.

Remote Viewer Behavior

If a Fast Command does not appear to run inside an RDP, VNC, or NoMachine session, check the clipboard and paste it manually into the remote desktop. That behavior is intentional.

DNS, ARP, or Route Data Looks Different Than Expected

The Network Inspector uses operating-system tools and psutil to collect data. Output detail can vary by platform and by the availability of route, ARP, or resolver utilities. A missing field does not always indicate a broken inspector; sometimes the OS simply does not report the same detail in the same format.

macOS Launch Warnings

Unsigned macOS builds can trigger Gatekeeper warnings on first launch. Use Finder's Open action if your security policy permits it.

Desktop Integration on Linux

If the app-menu entry goes missing or stops launching the current build, open Settings and run the desktop integration repair or reinstall action.

Unsure Which CLI Option to Use

Run snakesh --help, snakesh tool --help, or open the Command Line and Launchers page from the Help index. That reference lists the normal GUI launch modes, diagnostics flags, desktop-integration actions, standalone tool keys, Ping prefill options, and the internal helper flags that SnakeSh uses for child processes.

Session List Is Hidden

If the saved-session tree seems to disappear, check the dropdown beside the session search box. Auto mode intentionally collapses the panel until you hover the main splitter handle, and Float moves it into a separate always-on-top window. Switch back to Show if you want the tree pinned open in the main window.

Imported Configuration Is Missing Passwords

That is expected when the passwords or backend auth values live in the OS keyring or an external secrets backend. First-party exports carry the session/settings metadata, not the external secret material, so re-enter those values or reconnect the backend on the destination machine.

When to Use the Help Window

The Help window is non-modal. Keep it open beside SnakeSh while you work so session fields, settings descriptions, and tool usage stay visible without closing the current connection workflow.
