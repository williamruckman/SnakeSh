Fast Commands

Fast Commands store reusable command text that can be launched from the toolbar or menu without opening an external script editor. They are useful for routine operational commands, common diagnostics, and environment bootstrap snippets.

How Fast Commands Differ From the Bottom Command Bar

  • The bottom command bar is ad-hoc and always terminal-only.
  • Fast Commands are saved and reusable.
  • Fast Commands can adapt to the current target type, including remote viewer tabs.

Creating and Managing Commands

  • Use Add Fast Command... from the Fast Commands menu to save the current command-bar text or another multi-line payload.
  • Manage Fast Commands... opens the manager dialog for edit, rename, and delete actions.
  • Fast Commands are stored with application settings, so they can be included in normal settings export or import.

Terminal Tabs

When the active target is a terminal tab, SnakeSh injects the saved text into that terminal. If the tab belongs to a group, SnakeSh fans the same text out to every terminal in the group. Local Shell, SSH, Telnet, and other terminal-style tabs all follow this behavior.

Remote Viewer Tabs

When the active target is an RDP, VNC, or NoMachine viewer tab, SnakeSh copies the command text to the clipboard instead of trying to inject keystrokes. The copied payload does not include an extra trailing newline, so you can decide how and where to paste it inside the remote system.

Status Feedback

After a Fast Command is launched against a remote viewer tab, SnakeSh shows a status message telling you that the text was copied and should be pasted inside the remote session.

Practical Uses

  • Standard service restart commands
  • Environment-variable exports
  • Log tail commands
  • Inventory or health-check snippets
  • Paste-ready commands for remote desktops that do not expose a direct terminal channel
