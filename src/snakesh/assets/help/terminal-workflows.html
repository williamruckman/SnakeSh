Terminal Workflows

SnakeSh uses a shared terminal framework for SSH, Telnet, Serial, and Local Shell tabs. This keeps copy behavior, searchable scrollback, logging, shell banners, grouping, and detaching consistent across different connection types.

SSH and SFTP Workflow

SSH is the primary interactive terminal workflow. A typical sequence is:

  • Open an SSH tab for command-line work.
  • Launch an SFTP tab from the SSH context menu when file transfer is needed.
  • Use saved key authentication, password fallback, tunnels, automation, and logging as required.

Telnet Workflow

Telnet tabs are handled inside SnakeSh instead of delegating to an external program. This makes them suitable for legacy network devices or embedded targets where SSH is unavailable.

Serial Workflow

Serial sessions are useful for console servers, lab gear, appliances, and development boards. Because they use the same terminal view, you still get searchable scrollback and log capture instead of a minimal serial console.

Scrollback

Active terminal tabs expose a Scrollback button on the bottom command bar. The dialog opens the tab's retained terminal history in a separate non-modal window, so you can keep using the terminal while the viewer stays live as new output arrives.

  • Click Search or press Ctrl+F to open the find row.
  • Use Case for case-sensitive matching and Regex to search with regular expressions.
  • Use the next and previous buttons to move between matches; the match label shows the current index.
  • Press Escape to hide the find row and clear active search highlights.
  • When new output arrives while the dialog is open, the viewer text and any active search results refresh without forcing the view to jump to the newest line.

Local Shell Workflow

Local Shell tabs are launched from the main toolbar, not from the saved-session library. They are still part of the same workspace model as remote terminals. You can:

  • Detach and reattach them
  • Group them with other terminal tabs when that makes sense operationally
  • Include them in workspace profiles
  • Launch them with default or custom working-directory behavior

Automation and Reuse

Terminal-oriented sessions can run post-connect automation steps such as commands, sleeps, and expect-style waits. This is useful for banner handling, environment preparation, or jump-host sequences.

Shell Banners

SSH, Telnet, and Serial sessions can display a shell banner above the terminal with a custom color and optional blink. Use this for maintenance warnings, environment reminders, or other cues that should stay visible during the session.

Logging

Terminal logging can be enabled manually for a single tab or globally through Settings. Logs are stored using folder-aware paths so related sessions can stay organized on disk.

Copy and Keyboard Behavior

The terminal view keeps keyboard focus inside the remote session for keys such as Tab and Shift+Tab. Standard copy and paste behavior is available where appropriate, while control-key combinations still behave like terminal input when that is the safer default.

Double-click a visible terminal token to select the full contiguous non-whitespace word for quick copy from the context menu or keyboard shortcuts that use the current selection.

When pasted text contains multiple lines, SnakeSh opens a Review Multi-line Paste dialog before sending it to the terminal. The same review flow applies to the bottom command bar when a grouped or active terminal target would receive a multi-line batch.

Dropping local files onto a terminal view or the bottom command bar inserts full local paths in shell-safe single quotes, so commands keep working when filenames contain spaces or apostrophes.
