Settings and Themes

The Settings dialog controls the global behavior of SnakeSh. It covers appearance, terminal defaults, Local Shell behavior, logging, secrets backends, startup prompts, desktop integration, standalone tool launchers, and data migration tools.

Theme Presets and Colors

SnakeSh now defaults to the Onyx Blue theme. The default theme selection and the actual default color values are derived from the same preset so the label and colors stay in sync.

  • Choose a preset theme for quick styling changes.
  • Adjust terminal and tab colors manually if you need a custom palette.
  • Manual edits move the preset into Custom.
  • Restore Defaults returns theme-related values to the current default preset, which is Onyx Blue unless the application changes that default in a future release. Saving after a restore also clears saved Workspace Profiles, Fast Commands, Web Server Profiles, and Syslog / SNMP Monitor Profiles.

Window Behavior

Settings now includes the main-window fullscreen shortcut and an option to hide the top action bar and bottom command bar while fullscreen is active.

Terminal Behavior

  • Terminal font family and font size
  • Scrollback size for the retained searchable history viewer
  • Optional classic black/white default terminal colors for ncurses and TUI applications that depend on terminal defaults
  • Cursor blink
  • Optional BEL beep and visual bell flash
  • Active and inactive tab color styling

Local Shell Defaults

You can define the default shell command and choose where new Local Shell tabs start: your home directory, the application's current working directory, or a custom path.

Logging and Retention

Terminal session logging can be enabled manually per tab or globally through Settings. Separate cleanup policies can remove older session logs and archived web-server logs automatically after the configured retention windows, and crash logging can be turned on for fatal-error capture. The crash-logging toggle is intended for fatal faults only. For live freeze or hang investigations, launch SnakeSh with --debug-level debug so the run writes a dedicated diagnostics log.

Secrets Backend Setup

SnakeSh can store reusable credentials through multiple backends. The Settings dialog includes setup and test actions so you can validate the chosen backend before relying on it.

Startup and Platform Integration

  • Optional master-password prompt at startup
  • Optional master-password prompt before standalone tool launches
  • Default startup profile selection
  • Linux desktop integration install, repair, or removal
  • Cross-platform per-tool launcher install or removal through Manage Tool Launchers...
  • Import and export entry points for sessions and settings
