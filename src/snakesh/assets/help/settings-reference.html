Settings Reference

This page is the field-by-field reference for the Settings dialog. It covers the visible options, the provider-specific sections that appear conditionally, and the action buttons at the bottom of the dialog.

Appearance

  • Theme: chooses a preset palette. If you manually edit any theme color field, SnakeSh automatically switches the preset label to Custom.
  • App Background Start: start color of the main application gradient.
  • App Background End: end color of the main application gradient.
  • Text Color: default foreground text color used across the interface.
  • Field Background: background color for editable inputs, trees, tables, and similar widgets.
  • Field Border: border color for inputs and list-style controls.
  • Accent Color: base action color used by buttons and highlights.
  • Accent Hover: hover-state color for accent buttons.
  • Accent Pressed: pressed-state color for accent buttons.
  • Pick button beside each color: opens the color picker and previews the result live where supported.

Tab Colors

  • Tab Active Background: background color for the selected tab.
  • Tab Active Foreground: text color for the selected tab.
  • Tab Inactive Background: background color for non-selected tabs.
  • Tab Inactive Foreground: text color for non-selected tabs.

Window

  • Fullscreen Shortcut: keyboard shortcut that toggles borderless fullscreen for the main SnakeSh window.
  • Hide Controls in Fullscreen: when enabled, fullscreen hides the top action bar and bottom command bar so the session list and workspace remain.

Terminal

  • Terminal Background: default terminal canvas color used when classic default-color compatibility is disabled.
  • Terminal Foreground: default terminal text color used when classic default-color compatibility is disabled.
  • Classic Default Colors: maps the terminal's default colors to classic black and light gray for ncurses/TUI apps that rely on terminal defaults instead of explicit ANSI black.
  • Default Color Help: explains the difference between explicit ANSI colors emitted by an app and the terminal's own default foreground/background colors.
  • Terminal Font Family: fixed-width font used in terminal views.
  • Terminal Font (pt): terminal font size.
  • Terminal Scrollback Lines: retained terminal history line count available in the searchable scrollback viewer.
  • Session Log Folder: folder used for manual and global terminal session logging.
  • Global Session Logging: if enabled, terminal sessions begin logging automatically on connect.
  • Session Log Cleanup: enables automatic deletion of old session log files.
  • Session Log Retention (days): number of days to keep old terminal session log files when cleanup is enabled.
  • Web Server Log Cleanup: enables automatic deletion of archived web-server log files.
  • Web Server Log Retention (days): number of days to keep archived web-server log files when cleanup is enabled.
  • Crash Logging: records fatal-error details for troubleshooting. The inline Open Folder button opens the crash-log directory.
  • Cursor Blink: enables a blinking terminal cursor.
  • Terminal Bell: plays the terminal bell on BEL.
  • Visual Bell: flashes the UI on BEL instead of or in addition to audio feedback.
  • SFTP Delete Prompt: controls whether SFTP delete operations ask for confirmation.
  • SFTP Overwrite Prompt: controls whether SFTP uploads and downloads ask for confirmation before replacing existing files.
  • Active Tab Close Prompt: controls whether active terminal and remote-viewer tabs ask for confirmation before closing.

Local Shell

  • Default Shell Command: optional override for the command SnakeSh launches for new Local Shell tabs.
  • Startup Directory: chooses whether Local Shell starts in the home directory, the app's current working directory, or a custom path.
  • Custom Startup Directory: directory field used only when Startup Directory is set to Custom path.

Startup Security

  • Require a master password before opening SnakeSh: turns on the startup gate when a stored master password exists.
  • Tool Launch Password: row that contains Require a master password before launching standalone tools. It uses the same stored master password material and can be enabled independently from the main startup prompt once a master password exists.
  • New Password: new master password field.
  • Confirm Password: confirmation field for the new master password.
  • Clear Saved Password: row that contains the Clear stored master password checkbox used to remove the saved master-password hash material when settings are saved.
  • Master Password Help: inline status and guidance label that explains the current master-password state.

Secrets Backend

  • Backend Provider: chooses the credentials backend. Supported options are OS Keyring, 1Password CLI, Bitwarden CLI, Keeper Commander CLI, KeePass 2.x via KeePassXC CLI, and HashiCorp Vault KV v2.
  • Backend Auth: static reminder that sensitive backend tokens are stored in the OS keyring and blank fields keep the current values.
  • Backend Guide: dynamic setup help that changes with the selected provider.

1Password Provider Fields

  • 1Password Vault: target vault name.
  • 1Password Account: account shorthand or account domain.
  • 1Password CLI Path: path to the op executable.
  • 1Password Service Token: optional service-account token stored in the OS keyring. The nested Clear checkbox removes the stored value.

Bitwarden Provider Fields

  • Bitwarden CLI Path: path to the bw executable.
  • Bitwarden Session: optional BW_SESSION value stored in the OS keyring. The nested Clear checkbox removes the stored value.

Keeper Provider Fields

  • Keeper CLI Path: path to the keeper executable.
  • Keeper User: Keeper account username.
  • Keeper Master Password: optional master password stored in the OS keyring. The nested Clear checkbox removes the stored value.
  • Keeper Server (optional): optional Keeper server override.
  • Keeper Folder: target folder path used for SnakeSh-managed secrets.

KeePass Provider Fields

  • KeePass CLI Path: path to keepassxc-cli.
  • KeePass Database: path to the .kdbx database file.
  • KeePass Master Password: optional master password stored in the OS keyring. The nested Clear checkbox removes the stored value.
  • KeePass Password Env Var: environment variable name used as an alternative password source.
  • KeePass Key File (optional): optional key file path used with the database.
  • KeePass Group: target group path inside the database.

Vault Provider Fields

  • Vault Address: Vault base URL.
  • Vault Mount: KV v2 mount path.
  • Vault Token: optional token stored in the OS keyring. The nested Clear checkbox removes the stored value.
  • Vault Token Env Var: environment variable name used as an alternative token source.
  • Vault Namespace: optional Vault Enterprise namespace.
  • Vault TLS: Skip TLS certificate verification toggle.

Secrets Backend Actions

  • Setup Backend: runs backend-specific setup or readiness checks.
  • Test Backend: tests connectivity and configuration without changing stored secrets.
  • Backend Status: inline success or failure message from the most recent setup or test action.
  • Backend Test: static hint explaining that setup and test actions do not change secrets by themselves.

Bottom-of-Dialog Actions

  • Export SnakeSh: opens the first-party export flow for settings, sessions, or both.
  • Import SnakeSh: opens the first-party import flow.
  • Third Party Import: opens the external import workflow for SecureCRT, OpenSSH config, and PuTTY data.
  • Install/Repair Desktop Integration: Linux-oriented desktop launcher repair or install action when supported by the current build.
  • Remove Desktop Integration: removes the Linux desktop launcher integration and managed tool launchers when supported by the current build.
  • Manage Tool Launchers...: installs or removes per-user launcher entries for standalone tools on supported platforms. The --remove-tool-launchers command removes all managed standalone tool launchers without changing SnakeSh data.
  • Restore Defaults: resets fields to the app defaults. The warning notes that saving after this also clears saved Workspace Profiles, Fast Commands, Web Server Profiles, and Syslog / SNMP Monitor Profiles.
  • OK: validates the current dialog state and saves the new settings.
  • Cancel: closes the dialog without saving changes from the current editing session.
